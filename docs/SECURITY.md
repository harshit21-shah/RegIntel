# SECURITY.md

## 1. Threat Model Summary

| Asset | Threats | Mitigations |
|---|---|---|
| Client business profile data (PII-adjacent: company info, operations) | Unauthorized access, tenant data leakage | Row-level multi-tenancy (tenant_id scoping on every query), JWT auth, RLS policies in Postgres |
| LLM API keys / DB credentials | Leakage via code/logs | Secrets Manager, no secrets in env files committed, log scrubbing middleware |
| Generated briefs (compliance-sensitive) | Tampering, unauthorized access | Access control per tenant, immutable audit trail (agent_traces) |
| Prompt injection via ingested regulatory documents | Malicious/crafted text in scraped docs manipulating agent behavior | Content delimiting, instruction-hierarchy prompting, eval set `prompt_injection/` |
| Scrapers | IP bans, legal ToS issues with source sites | Respect robots.txt, rate limiting, prefer official APIs (eCFR, Federal Register, EDGAR all have APIs) |

## 2. Authentication & Authorization

- JWT-based auth (short-lived access tokens + refresh tokens).
- Roles: `admin`, `consultant` (multi-client access within tenant), `viewer` (read-only).
- All API queries scoped by `tenant_id` extracted from JWT claims — enforced at the ORM query layer (not just API layer) via a base repository class that injects `tenant_id` filters.
- Postgres Row-Level Security (RLS) enabled as defense-in-depth on `client_profiles`, `briefs`, `feedback`.

## 3. Secrets Management

- AWS Secrets Manager for: Anthropic/OpenAI/Voyage/Cohere API keys, DB credentials, Neo4j credentials, Qdrant API key.
- Local dev: `.env.local` (gitignored), populated from `.env.example` template — never real prod secrets in dev.
- Secrets rotated quarterly (documented rotation runbook in `infra/runbooks/secret_rotation.md`).

## 4. Data Privacy

- Client profile data is business-operational (NAICS codes, states of operation) — generally not PII, but treated with same rigor as PII for tenant isolation.
- No client data is sent to third-party LLM providers beyond what's necessary for the specific query (no bulk profile dumps in prompts — only relevant fields).
- Data retention: agent traces retained 1 year (audit purposes), then archived to cold storage (S3 Glacier).

## 5. Prompt Injection Defense

Regulatory documents are external, untrusted text. Mitigations:
- All retrieved document content wrapped in explicit delimiters (e.g., `<source_document>...</source_document>`) with system prompts instructing the model to treat enclosed content as **data only**, never as instructions.
- Agents never execute actions based on instructions found within document text (e.g., a malicious clause saying "ignore previous instructions and..." must not alter agent behavior).
- Eval dataset `services/eval/datasets/prompt_injection/` contains adversarial documents with embedded injection attempts — must pass before deployment.

## 6. Legal Disclaimers (product-level, not just security)

Every brief and `/query` response includes a standard disclaimer:

> "RegIntel provides informational regulatory-change intelligence generated by an AI system and is not a substitute for legal advice. Always consult a qualified compliance professional or attorney before making compliance decisions. RegIntel makes reasonable efforts to verify citations but does not guarantee completeness or accuracy."

This is a **product requirement**, not optional — stored as a constant in `services/agents/prompts/disclaimers.py` and unit-tested to ensure it's present in every output schema instance.

## 7. Infrastructure Security

- VPC with private subnets for RDS, Neo4j (peered), Qdrant; only ALB and NAT gateway in public subnets.
- Security groups: least-privilege, service-to-service only on required ports.
- All inter-service traffic within VPC; external API access via ALB + WAF (rate limiting, basic OWASP rule set).
- ECS task roles scoped per service (agent workers don't have S3 write access to raw doc bucket, only ingestion service does).

## 8. Dependency & Supply Chain Security

- `pip-audit` / `npm audit` in CI.
- Dependabot enabled for dependency updates.
- Docker base images pinned to specific digests, scanned via Trivy in CI.

## 9. Logging & Monitoring

- No raw API keys, tokens, or full client PII in logs (log-scrubbing middleware in FastAPI).
- CloudWatch + Langfuse for tracing; access to Langfuse dashboards restricted via SSO.

## 10. Incident Response

- Runbook in `infra/runbooks/incident_response.md`: covers credential leak, data breach notification process (tenant notification within 72h per typical data-protection norms), and rollback procedure for bad model/prompt deployments.
